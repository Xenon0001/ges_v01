# 📘 NOTA GUÍA PARA DISEÑADOR UI/UX — GES

## 1️⃣ Principio general

GES debe transmitir:

* Institucionalidad
* Claridad
* Orden
* Confianza
* Profesionalismo

No debe parecer:

* App juvenil
* Software experimental
* Sistema recargado

---

## 2️⃣ Filosofía visual

Inspiración:

* Sistemas ERP
* Dashboards administrativos
* Software bancario

No redes sociales.

---

## 3️⃣ Paleta de colores

Base institucional recomendada:

* Azul oscuro (confianza)
* Gris neutro
* Blanco limpio
* Verde suave para estados positivos
* Rojo sobrio para alertas

Evitar:

* Colores saturados
* Gradientes exagerados
* Neon

---

## 4️⃣ Tipografía

* Fuente clara
* Tamaños consistentes
* Jerarquía clara:

  * Título grande
  * Subtítulo medio
  * Texto normal
  * Texto secundario gris

---

## 5️⃣ Layout general

### Estructura recomendada:

```
┌ Sidebar fija
│
├ Header superior
│
└ Área principal de contenido
```

Sidebar:

* Íconos + texto
* Separación clara por módulos
* Indicador visual del módulo activo

Header:

* Nombre del centro
* Usuario logueado
* Botón configuración

---

## 6️⃣ Reglas UX

✔ Formularios claros
✔ Validación visual inmediata
✔ Botones primario/secundario diferenciados
✔ Confirmaciones antes de eliminar
✔ Feedback visible al guardar

Nunca:

✖ Popups innecesarios
✖ Demasiados modales
✖ Tablas saturadas

---

## 7️⃣ Tablas (muy importante)

* Encabezado fijo
* Filtrado arriba
* Búsqueda rápida
* Espaciado generoso
* No más de 10–15 columnas visibles

---

## 8️⃣ Dashboard

Debe mostrar:

* Total estudiantes
* Pagos del mes
* Morosidad
* Alertas académicas

Con tarjetas limpias.
No gráficos innecesarios en MVP.

---

## 9️⃣ Sensación final que debe generar

Cuando alguien lo vea debe pensar:

> “Esto parece software profesional importado.”

No:

> “Esto parece un proyecto de estudiante.”
